# LANClipSync 操作手册

## 快速开始

### 1. 获取软件

从发布页面下载 `LANClipSync.exe`，放到任意文件夹即可。不需要安装。

### 3. 首次使用流程

**两台电脑都需要操作：**

1. **输入共享密钥（可选）**
   - 如果希望传输加密，在两台电脑上输入相同的密钥（8-32 个字符）
   - 不输入密钥则不加密

2. **点击「▶ 点击启用同步」**

3. **等待自动连接**
   - 两台电脑在同一局域网下会自动发现对方
   - 在线设备列表中会显示对方
   - 点击设备名称 →「连接选中设备」

4. **开始使用**
   - 同步模式下：在一台电脑上复制内容，另一台直接 Ctrl+V
   - 手动模式下：点击「📤 发送剪贴板」或「📥 获取对端剪贴板」

## 详细操作

### 启动与停止

| 操作 | 方法 |
|------|------|
| 启动同步 | 点击「▶ 点击启用同步」按钮 |
| 停止同步 | 点击「⏹ 点击停止同步」按钮 |
| 退出程序 | 关闭窗口 |

### 连接设备

**自动发现（推荐）**
两台电脑启动同步后，在线设备列表会自动显示对方。选中设备点击「连接选中设备」即可。

**手动直连**
如果自动发现不生效（如跨网段），可以：
1. 在对端电脑上查看 IP 地址（日志区会显示 `本机IP: xxx.xxx.xxx.xxx`）
2. 在本端输入框输入 IP 地址
3. 点击「直连」

如果对端使用了非默认端口，可以输入 `IP:端口号` 格式，例如 `192.168.1.100:37022`。

### 刷新网卡列表

如果切换了网络（如从办公室 WiFi 切到手机热点），UDP 广播地址可能发生变化，需要手动刷新：

1. 点击「🔄 刷新网卡」按钮
2. 程序会重新枚举网卡接口并更新广播地址
3. 设备列表会自动刷新，再次发现对端

**无需频繁刷新**。程序在启动时自动获取一次即可正常工作，切换网络后再手动刷新。

### 模式切换

**同步模式**
适合日常使用。检测到剪贴板变化后自动发送给对端。

**手动模式**
适合需要精确控制的场景。需要手动点击：
- 「📤 发送剪贴板」— 将本机当前剪贴板内容发送给对端
- 「📥 获取对端剪贴板」— 请求对端发送它的剪贴板内容

**切换模式需要在停止同步状态下进行。**

### 类型过滤

勾选/取消勾选即可控制哪些类型的内容可以同步：

| 勾选 | 行为 |
|------|------|
| ☑ 文本 ☑ 图片 ☑ 文件 | 全部同步（默认） |
| ☑ 文本 ☐ 图片 ☑ 文件 | 只同步文本和文件，不同步图片 |
| ☐ 文本 ☐ 图片 ☐ 文件 | 全部不同步（等同于关闭同步） |

### 隐私模式

开启后，每次检测到剪贴板变化时会弹窗确认：

```
┌─ 隐私确认 ──────────────────────┐
│ 检测到 文本 变更                 │
│ 是否发送到对端？                 │
│ ┌──────────────────────────────┐ │
│ │ "Hello World"                │ │
│ └──────────────────────────────┘ │
│                    自动取消剩余 25 秒│
│                    [❌ 取消] [✅ 发送]│
└───────────────────────────────────┘
```

- 点击「✅ 发送」：发送到对端
- 点击「❌ 取消」：取消发送
- 不操作：30 秒后自动取消

### 剪贴板历史

历史记录显示所有接收到的剪贴板内容：

- **双击**历史条目 → 恢复该内容到本机剪贴板
- 鼠标悬停可查看完整预览
- 历史按类型独立限额（文本 50 条 / 图片 10 条 / 文件 20 条），超出后自动删除最旧的

### 文件传输

收到文件时：

1. 文件自动保存到缓存目录（默认 `%TEMP%/LANClipSync_cache/`）
2. 同时写入本机剪贴板
3. 在任意位置 Ctrl+V 即可粘贴出文件
4. 日志区会显示「📂 打开文件夹」链接（5 秒后消失），点击可打开保存目录
5. 日志区最多保留最近 2000 行，超出后自动删除最旧的行，避免内存占用过大

**更改文件保存位置**：
编辑 exe 同目录下的 `LANClipSync_settings.json`，修改 `cache_dir` 字段：

```json
"cache_dir": "D:/MyShareFolder"
```

### 密钥管理

输入密钥后点击启动，密钥自动生效。

- 密钥需要**两端一致**才能通信
- 不输入密钥 = 不加密传输
- 密钥支持 8-32 个字符
- 点击 👁 按钮切换密钥显示/隐藏

## 配置文件说明

配置文件 `LANClipSync_settings.json` 位于 exe 同目录下，程序首次运行或修改设置后自动生成。

### 手动修改示例

```json
{
  "device_name": "我的台式机",
  "cache_dir": "D:/Cache",
  "filter_text": true,
  "filter_image": false,
  "filter_files": true,
  "privacy_mode": true,
  "window_width": 800,
  "window_height": 900
}
```

**注意**：路径中的反斜杠需要转义为 `\\`，或者使用正斜杠 `/`。正确的写法：

```json
"cache_dir": "D:\\MyCache"      ← 正确
"cache_dir": "D:/MyCache"       ← 正确（推荐）
"cache_dir": "D:\MyCache"       ← 错误（\M 是非法转义）
```

## 注意事项

1. **防火墙**：如果连接不上，检查 Windows 防火墙是否阻止了程序。需要放行 TCP/UDP 端口 37020-37031。
2. **同一网段**：两台电脑需要在同一个局域网下（同一路由器 / 同一交换机）。
3. **多台设备**：当前版本只支持 1 对 1 连接，不能同时连接多台设备。
4. **不要重复运行**：在一台电脑上不要同时运行多个 LANClipSync 实例。
