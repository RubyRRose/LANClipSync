# LANClipSync 常见问题

## 连接问题

### 两台电脑互相发现不了

检查以下事项：

1. **同一局域网**：两台电脑是否连接同一个路由器/交换机？
2. **防火墙**：Windows 防火墙可能阻止了 UDP 广播。尝试暂时关闭防火墙测试，如果可以连接，请将 `LANClipSync.exe` 添加到防火墙白名单。
3. **端口占用**：如果默认端口 37020/37021 被占用，程序会自动尝试 +1 端口。如果其他程序占用了大段端口，可能连接失败。
4. **跨网段**：如果两台电脑在不同子网（如 192.168.1.x 和 192.168.2.x），自动发现不可用，需要使用「手动直连」功能输入 IP 连接。

### 连接上但无法同步

1. 检查**类型过滤**设置是否关闭了相关类型（如过滤掉了图片）
2. 如果开启了**隐私模式**，确认是否点错了「取消」
3. 检查密钥是否一致（如果设置了密钥）

### 连接经常断开

- 对端电脑可能进入了睡眠/休眠状态
- 网络不稳定
- 程序会自动重连，稍等片刻即可恢复

### 切换网络后找不到设备

切换 WiFi 或连接手机热点后，网卡接口和广播地址会变化。在界面上点击「🔄 刷新网卡」按钮，程序会重新枚举网卡并更新广播地址。

## 使用问题

### 同步延迟大

文件传输（尤其是大文件）需要时间，传输期间存在正常延迟。文本和图片同步通常 < 1 秒。

### 粘贴时文件内容不对

文件传输完成后需要等待进度条走完。进度条消失后才表示传输完成并写入剪贴板。

### 图片历史恢复不了

双击图片历史条目时，程序会从磁盘缓存读取图片文件。如果缓存文件已被清理（超过限额或手动删除），恢复会失败并显示错误消息。

### 日志文件越来越大

日志按天轮转，每天一个文件，保留 30 天。界面上的日志区保留最近 2000 行，超出后自动清理旧行。如果磁盘空间紧张，可以手动删除 `LANClipSync_*.log` 旧文件。

## 配置问题

### 修改了 settings.json 但没生效

**最可能的原因**：JSON 格式有误。特别是 Windows 路径中的反斜杠需要转义：

```json
// ❌ 错误：\P、\u、\t 是非法转义
"cache_dir": "D:\Python\uv\clibord\dist\temp"

// ✅ 正确：反斜杠写成 \\ 或使用正斜杠
"cache_dir": "D:\\Python\\uv\\clibord\\dist\\temp"
"cache_dir": "D:/Python/uv/clibord/dist/temp"
```

修改后保存，重启程序生效。

### 找不到 settings.json

文件在 `LANClipSync.exe` 同目录下。如果你是通过命令行运行的（`pixi run python lanclip_v10_run.py`），文件会在 Python 环境目录下。

建议直接用 exe 运行。如果通过命令行运行，入口为 `lanclip_v11_run.py`。

### 如何重置所有设置

退出程序，删除 exe 同目录下的 `LANClipSync_settings.json`，重新启动程序即可恢复默认设置。

## 技术问题

### 提示"无法启动，端口被占用"

程序会自动尝试 +1 端口（最多 10 次）。如果全部被占用，会启动失败。检查是否有其他程序占用了 37020-37031 端口。

### 传输大文件时崩溃

单条消息 payload 上限 100MB。超过此上限的文件传输可能失败。建议分批传输。

### 和某某软件冲突吗

LANClipSync 使用 Windows 标准剪贴板 API，理论上不会和常规软件冲突。但如果其他软件也在持续监听/改写剪贴板，可能出现竞态条件。

已知可能冲突的情况：
- 剪贴板增强工具（如 Ditto、ClipX）
- 远程桌面（RDP）的剪贴板重定向
- 虚拟机软件的剪贴板共享

### 杀毒软件报毒

PyInstaller 打包的单文件 exe 有时会被误报。这是打包工具的特征，不是程序本身的问题。可以添加信任或从源代码自行构建。

## 构建相关问题

### 我想自己从源码构建

```bash
# 需要安装 pixi
pixi run build
# 输出在 dist/LANClipSync.exe
```

### 源码依赖什么

Python 标准库，零第三方依赖。只需要 Python 3.11 和 PyInstaller（用于打包 exe）。
